# 運転月報 印刷時のページ番号フッター 設計書

## 背景・目的

運転月報画面の「印刷／PDF」出力時、各ページにページ番号が表示されず、複数月分をまとめて印刷した際などにページの順序や区切りが分かりにくい。印刷・PDF出力時のみ、各ページの末尾にページ番号を表示できるようにする。

## 対象範囲

「印刷／PDF」出力のみを対象とする。「Excelとして出力」「JSONへ出力」は対象外とし、現状のまま変更しない。

## 動作仕様

- 現在の運転月報は、CSSの`break-before: page`により常に2ページ構成(1ページ目: 1〜15日の運転記録表+15日点検、2ページ目: 16〜31日の運転記録表+走行距離等の合計+末日点検)に固定されている。ページ数が可変になることは無い。
- 各ページの末尾に「1 / 2」「2 / 2」というページ番号を表示する。
- このページ番号は、画面上でのプレビュー表示時には表示せず、印刷・PDF出力時にのみ表示する(画面表示ではページという区切りが視覚的に存在しないため)。

## 実装アプローチ

- `public/report.js`の`renderReportView`内、1ページ目の内容(`reportBlock(record.days, 1, 15, ...)` + `checklistBlock('点検日15日', ...)`)の末尾に`<p class="print-page-number">1 / 2</p>`を追加する。
- 同様に、2ページ目の内容(`.report-page2`のdiv内、`checklistBlock('点検日は月の末日', ...)`の末尾)に`<p class="print-page-number">2 / 2</p>`を追加する。
- `public/style.css`に、通常時は非表示、印刷時(`@media print`)のみ表示するスタイルを追加する:
  ```css
  .print-page-number { display: none; }
  @media print {
    .print-page-number { display: block; text-align: center; font-size: 0.8rem; margin-top: 0.5rem; }
  }
  ```
- ページ数は固定(常に2ページ)であるため、CSSカウンター等による動的計算は行わず、「1 / 2」「2 / 2」という固定文字列をそれぞれのページの末尾に直接記述する。

## スコープ外

- 「Excelとして出力」「JSONへ出力」機能への変更は行わない。
- ページ数が可変になるような将来のレイアウト変更(今回はスコープ外、現状のレイアウトが前提)。
