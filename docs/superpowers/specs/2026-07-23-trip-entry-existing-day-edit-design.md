# 運転記録入力の既存データ編集 設計書

## 背景・目的

現在、運転記録入力画面で日付を選び、その日に既に記録(出庫時メーター指針・行先・運転者・アルコールチェック)がある場合、保存しようとした時点で初めて「既に入力済みです。上書きしますか?」という確認ポップアップが表示される。既存の内容は画面上に表示されず、修正したい場合も一度全項目を入力し直す必要がある。

この課題を解消するため、車両・日付を選んだ時点で既存データがあれば自動的に入力欄へ反映し、その場で内容を確認・修正して保存できるようにする。

## 対象範囲

運転記録入力欄(出庫時メーター指針・行先・運転者・アルコールチェック)のみを対象とする。給油入力欄(給油量)は対象外とし、現状のまま変更しない。

## 動作仕様

- 車両・日付のいずれかを変更した瞬間(初回表示時の既定日付=今日を含む)、選択中の車両とその日付の組み合わせに既存データがあれば、自動的に入力欄(出庫時メーター指針・行先・運転者・アルコールチェック)へ反映する。
- 既存データがある場合、フォーム上部に「この日は既に入力済みです。内容を修正して保存できます」という注記を表示する。新規入力(その日にまだ記録がない)の場合は何も表示しない。
- 保存ボタンの文言(「この記録を保存」)は、既存データの有無にかかわらず変えない。
- 保存時に表示されていた「既に入力済みです。上書きしますか?」という確認ポップアップは廃止する。保存処理自体のマージロジック(`saveTripDay`が該当日のデータを部分的にマージして保存する仕組み)は変更しない。

## 実装アプローチ

- `public/trip-entry.js`にモジュール変数を2つ追加する: `tripSelectedDate`(現在選択中の日付。既定値は今日の日付)、`tripSelectedVehicleId`(現在選択中の車両ID。既定値はnullで、この場合はQRロック車両または一覧の先頭車両など既存の自然な既定動作に従う)。
- 日付欄の`change`イベント、および車両選択ドロップダウンの`change`イベントで、それぞれ`tripSelectedDate`/`tripSelectedVehicleId`を更新したうえで`renderTripEntryView()`を呼び直す。社有車/私有車の切替ボタンは既存通り再描画するが、その際`tripSelectedVehicleId`をnullに戻す(切替後の車両一覧の先頭に従わせるため)。
- `tripFormHtml()`内で、実際に使う日付・車両IDを`tripSelectedDate`/`tripSelectedVehicleId`(未設定ならQRロック車両または一覧の先頭車両)から決定し、該当する月報レコードの該当日データを`loadMonthlyLog`経由で参照する。データが存在すれば(`dayHasData`相当の判定)、各入力欄の`value`にその内容を差し込み、注記文を表示する。
- 車両選択ドロップダウンの`selected`判定に`tripSelectedVehicleId`も反映させる。これは、日付変更のたびに画面を再描画するようになることで、これまで(車両一覧の先頭が常に選択された状態に戻ってしまう)よりも高頻度でこの再描画が起こるようになるため、車両選択が意図せずリセットされないようにするために必要な修正である。
- 保存成功後は、`tripSelectedDate`を今日の日付に、`tripSelectedVehicleId`をnullに戻す(既存の「保存後は今日の日付に戻る」という挙動を維持する)。
- `onTripEntrySubmit`内の「上書きしますか?」確認(`confirm(...)`)呼び出しを削除する。それ以外の保存処理(`saveTripDay`呼び出し、日常点検イベントの判定、Firebaseへの自動送信)は変更しない。

## スコープ外

- 給油入力欄(給油量)は対象外。現状の挙動を維持する。
- 運転月報画面(`report.js`)や、既存のExcel/JSON手動取込・競合解決機能への変更は行わない。
- QRコードで車両が固定されている場合の全体挙動(選択肢を隠し固定表示する等)は変更しない。QR固定車両であっても、既存データの自動反映・修正は同様に働く。
