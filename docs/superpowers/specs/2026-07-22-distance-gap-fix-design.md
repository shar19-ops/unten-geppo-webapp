# 走行距離計算のギャップ対応 設計書

## 背景・目的

運転月報の走行距離は、対象日の翌日の出庫時メーター指針との差分で計算している(`computeDistance`)。そのため、休日等で車両を使わず翌日のメーター指針が未入力の場合、対象日の走行距離が空欄のままになってしまう。翌日以降、次にメーター指針が記録されている日まで遡って差分を取れば本来は計算できるため、この制限を解消する。

## 変更内容

`public/app.js`の`computeDistance(days, day)`関数を、「翌日固定」から「その日以降で最初にメーター指針が記録されている日(月末=31日まで)を探して差分を取る」方式に変更する。

- 対象日自体にメーター指針が無い場合は、従来通り空欄。
- 対象日以降、月末までどの日にもメーター指針が見つからない場合も、従来通り空欄。
- 月をまたいだ探索は行わない(月報レコードは月ごとに独立したデータのため)。
- `computeTotals`(合計走行距離の計算)は変更しない。空欄の日はスキップして数値のみ合計する既存ロジックのままで、ギャップを飛び越えた分は打ち消し合い(テレスコーピング)により二重計上されない。

**変更後のコード:**
```js
function computeDistance(days, day) {
  const cur = days[day];
  if (!cur || cur.meterReading == null) return '';
  for (let d = day + 1; d <= 31; d++) {
    const next = days[d];
    if (next && next.meterReading != null) {
      return next.meterReading - cur.meterReading;
    }
  }
  return '';
}
```

## スコープ外

- Excel出力(`xlsx-export.js`)は対象外とする。距離の値はExcelテンプレート自体の数式(G列)で計算されており、同じ制限がある可能性があるが、今回は画面表示・印刷のみを修正する。
