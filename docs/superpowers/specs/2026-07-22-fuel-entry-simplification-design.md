# 運転記録入力の給油UI簡素化 設計書

## 背景・目的

運転記録入力は運転前(出庫前)のみの記録であり、給油量はその時点ではまだ分からない。そのため「運転記録入力」フォーム内の「給油あり」チェックボックス+給油量入力欄は実際には使われない項目であり、削除する。給油量の記録は既存の「給油を後日記入」タブのみで行う運用とし、このタブ名を「給油入力」に改める。

## 変更内容

### 1. 運転記録入力フォームから給油関連UIを削除する(`public/trip-entry.js`)

`tripFormHtml()`内の「給油あり」チェックボックス(`#fuelToggle`)と、それに連動して表示される給油量入力欄(`#fuelField`)を削除する。`renderTripEntryView()`内でこれらの要素にイベントリスナーを登録している処理も削除する。

### 2. 保存時に既存の給油量を上書きしないようにする(`public/trip-entry.js`)

`onTripEntrySubmit`が構築する`dayData`オブジェクトから`fuelAdded`キーを完全に取り除く(`document.getElementById('fuelToggle').checked ? ... : null`という式ごと削除)。

`saveTripDay`(`storage.js`)は`record.days[day] = { ...record.days[day], ...dayData, ... }`という形で既存の日次データに`dayData`をマージする実装のため、`dayData`に`fuelAdded`キー自体が存在しなければ、既存の値(「給油入力」タブで記録済みの給油量、または`null`)がそのまま保持される。これにより、給油入力後に同じ日の運転記録(メーター・行先など)を編集・再保存しても、記録済みの給油量が消えなくなる(現状は`fuelAdded: null`で明示的に上書きされてしまう可能性があった)。

### 3. 「給油を後日記入」を「給油入力」にリネームする(`public/trip-entry.js`)

以下の箇所すべてを対象とする(`public/trip-entry.js`にのみ存在し、他ファイルに同文言の参照はない):
- モード切替タブのボタン文言
- 給油フォームの見出し(`<h2>`)
- コード内コメント(`tripEntryMode`の説明コメント、`給油の後日記入`セクションコメント)

### 4. 不要になったCSSを削除する(`public/style.css`)

`.fuel-field`ルール(給油量入力欄専用のスタイル)は、対応する要素が無くなるため削除する。

## スコープ外

- 車両選択を運転記録入力側と連動させる改修(以前保留にした「給油登録」改修)は本設計の対象外。
- 「給油入力」フォーム自体の項目(車両・日付・給油量)は変更しない。
