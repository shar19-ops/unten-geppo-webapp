# 車両リストの管理者パスワード保護 設計書

## 背景・目的

社有車・私有車の登録内容(車両リスト)は、誰でも自由に追加・編集・削除できてしまう状態になっている。誤操作や意図しない変更を防ぐため、参考にした「馬券投票アプリ」(`C:\Users\shar1\OneDrive\MCフォルダ\馬券投票アプリ\keiba-baren-v3.html`)と同じ方式で、簡易なパスワード保護をかける。

馬券投票アプリの方式:
- パスワードはJSコード内の定数と単純比較する、クライアント側のみの簡易な仕組み。
- 解除フラグは`sessionStorage`に保存し、ブラウザ(タブ)を閉じると解除状態は消える。
- ヘッダーに常時表示されたチェックボックス(馬券投票アプリでは「幹事」)をONにするとパスワード入力オーバーレイが開き、正しいパスワードで該当タブ(馬券投票アプリでは「レース」「設定」)が表示される。チェックを外すと即座に再ロックされる。

本アプリでもこの方式をそのまま踏襲し、「車両リスト」タブに適用する。

**セキュリティ上の注意:** パスワードはJSコード内に平文で埋め込まれるため、ブラウザの開発者ツール等で読み取ることが可能。これは「うっかり誰でも編集できてしまう」ことを防ぐための簡易的な仕組みであり、本格的なアクセス制御(サーバー側認証)ではない。馬券投票アプリと同じ前提・同じリスク許容度で採用する。

## 変更内容

### 1. ヘッダーに管理者チェックボックスを追加する(`public/index.html`)

`app-header`内に「管理者」チェックボックス(`id="adminModeCheck"`)を追加する。3タブの下部固定タブバーとは別に、ヘッダーに常時表示する。

### 2. パスワード入力オーバーレイを追加する(`public/index.html`)

馬券投票アプリの`.pw-overlay`/`.pw-dialog`と同様の、パスワード入力欄+「キャンセル」「解除」ボタンを持つオーバーレイを、デフォルト非表示(`hidden`属性)で追加する。

### 3. オーバーレイ・チェックボックスのスタイルを追加する(`public/style.css`)

既存の`.panel`/`.btn`系のスタイルに合わせた、オーバーレイの見た目(画面中央に固定表示するダイアログ)のCSSを追加する。

### 4. 管理者ロック制御ロジックを追加する(`public/app.js`)

- 定数`ADMIN_PASSWORD = 'anzen_kanri'`と、`sessionStorage`キー`ug_admin_unlocked`を定義する。
- ページ読み込み時、`sessionStorage`の解除フラグに応じて「車両リスト」タブボタン(`.tab-btn[data-view="vehicles"]`)の表示/非表示とヘッダーのチェックボックスのチェック状態を初期化する。
- チェックボックスをONにしたとき、まだ解除されていなければパスワードオーバーレイを表示する。
- オーバーレイの「解除」ボタン(またはEnterキー)で、入力値が`ADMIN_PASSWORD`と一致すれば: `sessionStorage`に解除フラグを保存し、オーバーレイを閉じ、「車両リスト」タブボタンを表示する。一致しなければ、エラーメッセージを表示し(既存の`.status.error`と同じスタイル)、入力欄を再選択状態にする。
- オーバーレイの「キャンセル」ボタン(またはEscapeキー)で、オーバーレイを閉じ、チェックボックスをOFFに戻す。
- チェックボックスをOFFにしたとき、`sessionStorage`の解除フラグを削除し、「車両リスト」タブボタンを非表示にする。その時点で表示中の画面が「車両リスト」だった場合は「運転記録入力」タブに切り替える。

## スコープ外

- サーバー側認証・ユーザーごとのアカウント管理は対象外(既存のパスワード比較のみの簡易方式を踏襲)。
- 「車両リスト」以外の画面への同様のパスワード保護は対象外。
