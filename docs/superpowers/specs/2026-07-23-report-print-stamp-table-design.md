# 運転月報 印刷時の押印欄 設計書

## 背景・目的

運転月報の印刷・PDF出力時、最終ページに「安全運転管理者」「副安全運転管理者」「発行者」の押印(または署名)欄が無く、紙で運用する際に手書きで欄を書き足す必要がある。最終ページの末尾に、この3項目の押印欄を印刷時のみ表示できるようにする。

## 対象範囲

「印刷／PDF」出力のみを対象とする。「Excelとして出力」「JSONへ出力」は対象外とし、現状のまま変更しない(Excel出力は既存テンプレートファイルのセル値のみを書き換える方式のため、テンプレート自体に押印欄が必要かは別途検討する)。

## 動作仕様

- 最終ページ(2ページ目)、末日点検表の下、ページ番号(「2 / 2」)の手前に、3列(安全運転管理者・副安全運転管理者・発行者)の押印欄を配置する。
- 表はヘッダー行(3つの見出し)と、押印用の空白行1行で構成する。
- このページ番号と同様、押印欄は画面上のプレビュー表示時には表示せず、印刷・PDF出力時にのみ表示する。

## 実装アプローチ

- `public/report.js`の`renderReportView`内、末日点検表(`checklistBlock('点検日は月の末日', record.checklistEnd)`)の直後、ページ番号(`<p class="print-page-number">2 / 2</p>`)の直前に、押印欄のテーブルを追加する。既存の`.report-table`クラスを流用し、他の表と罫線・フォントの見た目を揃える。
- `public/style.css`に、ページ番号と同じ考え方で、通常時は非表示、印刷時(`@media print`)のみ表示する新クラス`.print-stamp-table`を追加する:
  ```css
  .print-stamp-table { display: none; }
  @media print {
    .print-stamp-table { display: table; }
  }
  ```
- 押印用の空白行には、押印(または手書き署名)に十分な高さとして`height: 3rem`程度を確保する。

## スコープ外

- 「Excelとして出力」「JSONへ出力」機能への変更は行わない。
- 押印欄への氏名の印字(データとしての入力・保存)は行わない。あくまで空白の押印・署名欄を印刷するのみ。
