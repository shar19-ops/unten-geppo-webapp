# 車両リスト統合(社有車・私有車)と私有車QR配布 設計書

## 背景・目的

現状、社有車は「社有車リスト」画面で事前登録し、私有車は運転記録入力画面でその都度自由テキスト入力する運用になっている。

実際の運用では、私有車の使用は紙ベースの「私有車使用届」が管理者に回送されてくる形で発生する。この設計では、届出が回送されてきた時点で管理者が私有車も事前登録し、その車両専用のQRコードを運転者へ配布する運用に変更する。これにより、社有車・私有車のどちらも「登録済みの車両をQRで読み取って運転記録入力する」という同じ体験に統一する。

本設計は、既にmainへのマージ待ちであるPR #1(社有車のQRコード自動選択機能)の上に直接積み増す形で実装する(同一ブランチ`worktree-qr-vehicle-select`)。

## データモデル(`storage.js`)

車両マスタ(`vehicles`配列)に以下を追加する。

- `vehicleType`: `'company'`(社有車) | `'private'`(私有車)。既存レコードにこの項目が無い場合は`'company'`として扱う(後方互換、マイグレーション処理は不要)。
- `driverName`: 私有車専用の必須項目(使用者名本人)。社有車では未使用(空文字/表示なし)。

私有車も社有車と同じ`vehicles`配列・同じ`id`体系で登録する。`vehicleRefFor(vehicleId, privateCarLabel)`は、登録済み私有車についても(社有車と同様に)その車両の`id`を`vehicleId`としてそのまま使う。

**既存の自由入力による私有車の運転記録**(もしあれば)は、従来通り`private:<サニタイズ済みテキスト>`というrefのまま履歴として残る。登録後の新しい記録は新しい`vehicleId`ベースのrefになるため、同じ車であっても登録前後で履歴は連続しない。この仕様上の割り切りは確認済み。

`mergeVehicles`(Excel/JSON取込時の重複検出)は、`plateNumber`だけでなく`vehicleType`も合わせて一致判定する(異なるタイプの車両が同じ車両番号を持っていても別物として扱う)。比較対象フィールドは、社有車なら`defaultManager`、私有車なら`driverName`を使う。

## 車両リスト画面(`vehicles.js` / `index.html`)

- ナビゲーションタブの表示名を「社有車リスト」→「車両リスト」に変更する(内部の`data-view="vehicles"` / `#view-vehicles`は変更しない)。
- 画面内に「社有車」「私有車」のタブ(運転記録入力の車両選択と同じ見た目の`segmented`ボタン)を追加し、選択中のタブに応じて別々の一覧テーブルを表示する。
  - 社有車タブの列: 車両番号・車種／名称・事業所名・既定の車両管理者・状態・操作
  - 私有車タブの列: 車両番号・車種／名称・事業所名・**使用者名**・状態・操作
- 追加・編集フォームは選択中タブに応じて項目を出し分ける(私有車タブでは「既定の車両管理者」の代わりに「使用者名(必須)」を表示)。
- Excel取込・Excel出力・JSON取込・JSON出力は、社有車タブ・私有車タブの**両方**に用意する(私有車タブのExcel列は「使用者名」を含み「既定の車両管理者」を含まない)。取込・出力ロジックは既存の社有車用処理と同じ関数群を、選択中タブの`vehicleType`を渡して共通利用する形にする。
- QRコード表示・印刷ボタンは両タブの各行に表示する(社有車・私有車とも同じ`?vehicle=<id>`形式)。
- 画面下部の「※ 私有車は運転記録入力画面でその都度自由入力します(このリストには登録されません)。」という案内文は削除する。

## 運転記録入力画面(`trip-entry.js`)

- 私有車モードの自由テキスト入力(`<input name="privateCarLabel">`)を廃止し、社有車と同様に登録済み私有車から選ぶ`<select>`に変更する。私有車が1件も登録されていない場合は「私有車が未登録です。「車両リスト」画面で登録してください。」という案内を表示する。
- `resolveVehicleSelection`は、私有車モードでも選択した私有車の`vehicleId`を返すように変更する。`tripUsePrivateCar`は「社有車タブ/私有車タブのどちらから選ぶか」というUI上の選択状態としての意味だけになり、保存される運転記録自体は社有車・私有車とも同じ`vehicleId`ベースになる(`privateCarLabel`は新規記録では使用しなくなる)。
- QR自動選択(`tripQrVehicleId`)は、対象車両の`vehicleType`を見て、私有車であれば`tripUsePrivateCar = true`にした上でその私有車を選択する。

## QRコードの私有車への拡張(`vehicles.js` / `app.js`)

- 私有車タブの各行にも「QRコード」ボタンを表示し、社有車と全く同じ仕組み(`${location.origin}${location.pathname}?vehicle=<id>`)でQRを生成・印刷できるようにする。
- `app.js`の起動時パラメータ解析(`applyQrVehicleParam`)は、社有車・私有車を問わず`loadVehicles()`全体から`?vehicle=`のIDに一致する車両(`active !== false`)を探し、見つかった車両の`vehicleType`に応じて`tripUsePrivateCar`をセットする(現状は常に`false`固定になっているのを、タイプ判定に変更する)。

## 運転月報画面(`report.js`)への影響

- `reportVehicleOptions()`は、`loadVehicles()`から社有車・私有車の両方を候補として構築する(ラベルの「(私有車)」表示は`vehicleType`で判定する)。加えて、登録済み車両のidに一致しない過去の自由入力私有車履歴(`privateCarLabel`ベース)については、従来通りログ索引から拾って候補に残す(後方互換)。
- 事業所名は、私有車も車両マスタから転記表示されるようになる(社有車と同じ挙動)。「私有車には転記元が無いため空欄」という現状の仕様・コメント・案内文言は削除する。この挙動変更は確認済み。

## スコープ外

- 社有車・私有車リストの追加・編集・削除を管理者のみに制限する権限機能(認証機構が必要なため別プロジェクト、以前のPRから引き続き対象外)。
- 「給油登録」画面の改修(別途保留中)。
