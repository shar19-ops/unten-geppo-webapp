# QRコードによる車両自動選択 設計書

## 背景・目的

社有車は「個人専用」と「共用」に分かれる。共用車は不特定多数の社員が使うため、運転記録入力のたびに手動で車両を選ぶのは手間がかかり、選び間違いも起こりうる。各車両にQRコードを備え付け、運転する社員がiPhoneのカメラで読み取るだけで、運転記録入力画面にその車両が自動選択された状態で開けるようにする。

管理者のみが社有車リストを追加・編集・削除できるようにする権限周りの改修は、認証機構の新規導入が必要になり影響範囲が別物のため、本設計の対象外とし別途改めて設計する。

## 全体の流れ

1. 社有車リスト画面(`vehicles.js`)の各車両行に「QRコードを表示」ボタンを追加する。
2. ボタン押下で、その車両のID(既存の`v.id`)を埋め込んだURL
   `<現在のorigin+pathname>?vehicle=<車両ID>`
   のQR画像をその場で生成するパネルを開く。パネルには車両番号・車種名も併記し、印刷ボタンでA4用に印刷して車両に貼り付けられるようにする。
3. QR生成には外部npm依存を追加せず、ExcelJS/SheetJSと同じ方針で `public/vendor/qrcode/` に軽量な単一ファイルのQR生成ライブラリ(ビルド不要・ブラウザ完結)を配置して使う。
4. 社員がiPhoneのカメラでQRを読み取るとSafariが開き、上記URLへアクセスする。
5. アプリ起動処理(`app.js`)で `location.search` の `vehicle` パラメータを読み取り、該当する社有車が存在すれば:
   - 表示タブを「運転記録入力」に固定する(現状も既定表示のため実質変更なし)
   - 車両モードを「社有車」に固定する(`tripUsePrivateCar = false`)
   - 該当車両を車両選択プルダウンで自動選択した状態で表示する
6. 反映後、`history.replaceState` でURLから `?vehicle=` を除去し、アドレスバーを綺麗な状態に戻す。次回同じQRを読み取れば同じ処理が再度走るため、これによる不都合はない。

## データモデル

新規のデータモデル追加は不要。既存の社有車マスタ(`vehicles.js` / `storage.js` の `v.id`)をそのままQRの車両識別子として使う。QRは「車両IDを運ぶだけのラベル」であり、該当車両がリストから削除された場合はそのQRは単に「該当車両なし」として機能しなくなる(要望通り、削除されるまでそのまま使い続けられる)。

## エラー処理

`?vehicle=` のIDが現在の社有車リストに存在しない(削除済み・誤読み取りなど)場合は、通常通り車両選択画面を表示しつつ、「QRコードに対応する車両が見つかりませんでした。車両を選び直してください」という案内メッセージを表示する。

## 注意点・運用上の制約

- QR画像は生成時点のページのURL(origin)を基にするため、印刷用QRは必ず本番のGitHub Pages URL(`https://shar19-ops.github.io/unten-geppo-webapp/`)を開いた状態で発行する。ローカルの `http://localhost:5174` から発行すると、他の端末では使えないQRになる。
- 私有車には従来通りQRは発行しない(車両マスタに登録されないため対象外)。

## スコープ外(別設計とする)

- 社有車リストの追加・編集・削除を管理者のみに制限する権限機能(認証機構の新規導入が必要なため別プロジェクトとする)
- 「給油登録」画面の改修(別途保留中)
